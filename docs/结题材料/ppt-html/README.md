# STrans 验收答辩 HTML 母版 V1

## 1. 用途

本目录是 16:9 答辩 PPT 的可审阅 HTML 原型。它不是最终 PowerPoint 文件，而是用于逐步确认：

1. 叙事顺序是否符合初始要求和评分权重；
2. 每页的信息密度、图文比例和事实边界是否合适；
3. 真实截图、UML 图和测试数据放在哪一页；
4. 确认后再转换或重建为正式 PPT。

## 2. 当前视觉方案

- 主色：交通工程蓝，表示系统与主流程；
- 辅色：青色，表示数据、算法和正向结果；
- 风险色：橙色/红色，仅用于限制、告警和待改进项；
- 版式：白底、低阴影、8px 顶部状态线、统一页码和英文页脚；
- 字体：Microsoft YaHei / Noto Sans CJK SC；
- 目标：适合教室投影、截图清晰、打印为 16:9 PDF 时不溢出。

## 3. 16 页叙事顺序

| 页 | 内容 | 建议净讲述时间 |
|---|---|---:|
| 1 | 封面与项目定位 | 15 秒 |
| 2 | 初始任务与评分导向 | 30 秒 |
| 3 | 四类核心问题 | 35 秒 |
| 4 | 目标、范围与成果边界 | 25 秒 |
| 5 | 系统总体架构 | 40 秒 |
| 6 | 角色与功能用例 | 25 秒 |
| 7 | 车辆与车牌算法 | 40 秒 |
| 8 | 真实运行效果 | 25 秒 |
| 9 | 道路异常与失败案例 | 40 秒 |
| 10 | 道路建模与空间分析 | 35 秒 |
| 11 | 自适应调度与环境边界 | 30 秒 |
| 12 | 软件工程与数据闭环 | 35 秒 |
| 13 | 测试与验收证据 | 45 秒 |
| 14 | Git 算法演进 | 30 秒 |
| 15 | 成果与创新 | 30 秒 |
| 16 | 限制、结论和演示衔接 | 20 秒 |

净讲述约 8 分 20 秒，保留约 1 分 40 秒用于换页、视频播放和现场停顿，满足初始要求中的 10 分钟报告。

## 4. 本地预览

在仓库根目录启动静态服务器：

```powershell
python -m http.server 4174 --bind 127.0.0.1
```

打开：

```text
http://127.0.0.1:4174/docs/%E7%BB%93%E9%A2%98%E6%9D%90%E6%96%99/ppt-html/index.html
```

支持：

- 工具栏上一页/下一页；
- 方向键、PageUp/PageDown、空格、Home、End；
- 浏览器“打印”输出 16:9 PDF；
- 每页下方的讲述提示不会出现在打印结果中。

## 5. 逐步确认流程

### 第一步：确认主线与风格

重点看第 1–5 页：封面、要求、问题、范围和架构。确认项目名称、颜色和“问题—方案—证据”主线。

### 第二步：确认算法页

重点看第 7、9、10、11 页。确认算法深度、图中文字大小，以及道路异常“候选 + 复核”的表述。

### 第三步：确认效果与测试页

重点看第 8、13、15 页。核对真实截图、100 测试单元、65 帧与 12 路机位口径，补充有人工真值的数据后再写精度指标。

### 第四步：补齐正式答辩信息

填入团队成员、指导教师、学院、答辩日期；决定是否在第 8 页嵌入 `live12_multi_vehicle_6s.mp4`，并最终生成 PPT/PDF。
