## 3. 核心算法迭代逻辑

### 3.1 从单帧检测到时序稳定

初始目标是得到 YOLO 检测框，随后问题转向“小目标、遮挡、重框和短时漏检”。当前实现通过 ByteTrack ID、视觉 ID 恢复、框平滑、可信轨迹短时预测和 5 帧车辆数中值，把输出从单帧结果提升为稳定交通状态。

答辩表述：

> 我们发现沙盘目标尺寸小、遮挡频繁，单帧高置信度检测会漏车，单纯降低阈值又会引入树木和道路设施误检，因此采用“低阈值保召回 + 道路/外观过滤 + 多目标跟踪 + 时序平滑”的迭代路线。

### 3.2 从一次 OCR 到车辆—车牌联合识别

车牌模块由单图测试逐步演化为车辆裁剪 OCR、全帧 OCR 补充、车牌到车辆的几何关联、按轨迹记忆、多帧稳定和白名单决策。实时流引入 OCR 预算与轮转，解决多辆排队车中后车长期得不到 OCR 机会的问题。

答辩表述：

> 车牌不是独立识别后简单显示，而是与车辆轨迹绑定；稳定车牌才进入白名单判断，短时识别波动不会立即改变通行决策。

### 3.3 从像素热点到道路语义热力图

第一版移动视角热力图直接在画面上叠加检测热点，随后增加 SegFormer 道路掩膜，把热点限制在道路区域；固定视角则使用单应矩阵投影到统一道路模型，按活跃轨迹数量与速度生成车道/路口状态。

答辩表述：

> 热力图经历了“像素叠加—道路分割约束—道路模型投影”三步迭代，从视觉效果升级为具有交通空间含义的状态表达。

### 3.4 从简单帧差到可解释的道路异常融合

异常识别的早期主体是道路 ROI 内帧差。随后逐步增加：

1. YOLO/ByteTrack 解释车辆与行人；
2. 光流抑制全局相机运动；
3. 变化比例检测光照突变和移动视角；
4. 亮度、Lab 色差、HSV 暖色、纹理、熵、轮廓填充率和圆度联合过滤；
5. 车道线和箭头专项过滤；
6. 静态图片独立分支；
7. 连续多帧 IoU 确认；
8. 可选道路破损模型补充。

答辩表述：

> 与其把所有异常都交给一个缺少沙盘标注数据的端到端模型，我们采用可解释的多线索融合：模型负责解释正常道路使用者，几何和时序规则负责发现无法解释的局部变化。

### 3.5 从固定参数到资源感知调度

在算法功能稳定后，迭代重点转向工程运行。系统根据静态/实时来源、车辆/异常任务、CPU、内存、GPU、显存和推理耗时在质量、异常、实时、保护和均衡档间切换，并通过缓存与滞回防止频繁抖动。

答辩表述：

> 调度模块不改变算法结论的业务语义，而是动态选择模型、尺寸、阈值和检测间隔，使同一套系统可以在不同硬件负载下保持可用。

## 4. 提交历史反映的软件工程方法

- **先打通纵向闭环**：视频输入、后端、前端先形成可运行链路，再扩展算法。
- **算法与管理同步演进**：识别结果逐步进入 SQLite、告警、处置、证据和审计，而不是停留在演示脚本。
- **针对真实误差迭代**：车辆重框、后车车牌漏识别、非道路热点、异物误报都对应独立提交。
- **保留降级路径**：远程算法不可用时使用本地模型；资源紧张时降尺寸/频率；录像压缩失败时回退流复制。
- **增加可验证性**：道路掩膜、热力图、录像导出和告警证据包使结果可以复查。

## 5. 后续迭代建议

1. 建立带真值的沙盘车辆、车牌和异物测试集，提交时自动计算指标；
2. 把当前阈值集中为可版本化配置，记录每次评测使用的配置哈希；
3. 为道路异常增加事件级去重和持续时间统计；
4. 对固定摄像头分别标定速度区域，减少非参考视角的估计误差；
5. 在有足够数据后比较规则融合与监督模型，决定是否替换静态异常分支；
6. 为模型切换增加基准测试，让调度策略使用真实设备上的 P95 延迟而不仅是瞬时耗时。
