# M05：道路语义分割 — RoadMaskService

## 1. 模块定位

`RoadMaskService` (`backend/app/services/road_mask.py`，28 符号，139 行) 使用 SegFormer-B0 语义分割模型识别画面中的道路区域，输出二值掩膜和道路示意底图，用于约束移动视角的热力图——使热点仅在道路区域内绘制。

## 2. 模型选型

| 属性 | 取值 |
|---|---|
| 模型 | `nvidia/segformer-b0-finetuned-cityscapes-1024-1024` |
| 来源 | HuggingFace Hub (`snapshot_download`) |
| 本地路径 | `data/models/segformer-b0-cityscapes/` |
| 框架 | PyTorch + HuggingFace Transformers |
| 推理设备 | cuda:0 (优先) / cpu |
| 目标标签 | Cityscapes 中的 "road" 类 |

## 3. 架构设计

```
RoadMaskService
├── 模型加载 (_load_model)
│   ├── 检查 config.json 存在 → 否则 download_model()
│   ├── AutoImageProcessor.from_pretrained()
│   ├── AutoModelForSemanticSegmentation.from_pretrained()
│   ├── road_class_ids(id2label) → 提取 "road" 标签
│   └── model.to(device).eval()
│
├── 推理 (snapshot)
│   ├── BGR → RGB
│   ├── processor(images=frame_rgb)
│   ├── model(**inputs).logits
│   ├── interpolate → 原始尺寸
│   ├── argmax → label map
│   ├── np.isin(labels, road_ids) → 二值掩膜
│   ├── resize → max 256px 宽度 (节省带宽)
│   └── encode_mask_data_url → PNG base64
│
└── 缓存 (_cache)
    └── 按 camera_id, 0.5s TTL
       (语义分割较慢，避免每帧都推理)
```

## 4. 核心算法

### 4.1 道路类别提取

```python
def road_class_ids(id2label):
    return {int(index) for index, label in id2label.items()
            if str(label).strip().lower() == "road"}
```

Cityscapes 的 "road" 标签被提取为二值掩膜的目标类别。

### 4.2 推理管道

```python
# 1. 预处理
frame_rgb = cv2.cvtColor(frame, BGR2RGB)
inputs = processor(images=frame_rgb, return_tensors="pt")
inputs = {k: v.to(device) for k, v in inputs.items()}

# 2. 前向推理 (无梯度)
with torch.inference_mode():
    logits = model(**inputs).logits

# 3. 上采样到原图尺寸
logits = interpolate(logits, size=(H, W), mode="bilinear")

# 4. 取 argmax + 道路标签选择
labels = logits.argmax(dim=1)[0].cpu().numpy()
binary = np.where(np.isin(labels, road_ids), 255, 0).astype(np.uint8)

# 5. 降采样 (限制输出尺寸)
mask_w = min(W, 256)
mask_h = round(H * mask_w / W)
binary = cv2.resize(binary, (mask_w, mask_h), INTER_NEAREST)
```

### 4.3 道路示意底图

```python
def render_road_schematic(mask):
    binary = np.where(mask > 0, 255, 0).astype(np.uint8)
    canvas = np.full((h, w, 3), (25, 39, 52), dtype=np.uint8)  # 深色背景
    canvas[binary > 0] = (72, 93, 104)                          # 灰色道路
    # 道路轮廓线
    contours = findContours(binary, RETR_EXTERNAL, ...)
    drawContours(canvas, contours, -1, (117, 209, 224), line_width)
    return canvas
```

生成一个简洁的道路轮廓图作为热力图的底图（"示意图模式"）。

### 4.4 PNG Base64 编码

```python
def encode_mask_data_url(mask):
    ok, encoded = cv2.imencode(".png", mask)
    return "data:image/png;base64," + base64.b64encode(encoded).decode()
```

`data:` URL 格式可直接在 `<img>` 或 Canvas 中使用，无需额外 HTTP 请求。

## 5. 模型下载

```python
def download_model(self):
    MODEL_DIR.mkdir(parents=True, exist_ok=True)
    snapshot_download(repo_id=MODEL_ID, local_dir=MODEL_DIR)

def ensure_model_available(self):
    if not (MODEL_DIR / "config.json").exists():
        self.download_model()
```

首次启动时自动从 HuggingFace 下载，后续使用本地缓存。下载通过 `snapshot_download` 实现断点续传和哈希校验。

## 6. 缓存与线程安全

```python
class RoadMaskService:
    def __init__(self):
        self._lock = threading.Lock()
        self._cache: dict[camera_id → (timestamp, result)]

    def snapshot(self, camera_id, jpeg):
        # 双重检查锁定
        if cached and now - cached[0] < 0.5:
            return cached
        with self._lock:
            if cached and now - cached[0] < 0.5:
                return cached
            # 实际推理...
            self._cache[camera_id] = (now, result)
```

0.5 秒 TTL 意味着同一摄像头每秒最多做 2 次语义分割。

## 9. 设计考量

### 为什么是 SegFormer-B0？

- **轻量**：B0 是最小变体，适合本地 GPU 推理
- **Cityscapes 预训练**：城市道路场景高度匹配
- **1024×1024 分辨率**：处理好 1080p 画面

### 为什么限制输出到 256px 宽？

- 前端 Canvas 渲染不需要原始分辨率
- PNG base64 体积随分辨率线性增长
- 256px 足够指导热力点的剪裁精度
- `INTER_NEAREST` 保持二值掩膜的锐利边缘

### 为什么缓存 0.5s？

- 道路分割是全局操作，远比 YOLO 检测昂贵
- 沙盘摄像头为固定视角，道路形状不变
- 移动摄像头（手机）会有轻微差异，0.5s 刷新足够
