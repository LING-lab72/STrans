# M12：前端架构 — React 大屏 + 热力图 + 语音控制

## 1. 模块定位

前端 (`frontend/src/`) 是一个基于 React + Vite 的单页应用（SPA），采用监控大屏的设计风格。核心文件：

| 文件 | 职责 |
|---|---|
| `main.jsx` | 主入口：路由、状态管理、全部 UI 组件（约 2384 行） |
| `heatmap.js` | 热力图：热点坐标变换、检测框变换、渲染模式解析 |
| `cameraTypes.js` | 摄像头类型常量（解决 BUG-002） |
| `auth.js` | 验证码请求的 1/2/4 秒指数退避策略 |
| `roadModeler.js` | 道路建模同源路径和安全新标签页属性 |
| `public/road_logic_modeler/` | 项目内置道路建模 HTML/CSS/几何/模型/渲染/交互文件 |
| `styles.css` | 全局样式 |

## 3. 核心组件

### 3.1 MJPEG 双画面

```jsx
<img src={`/api/cameras/{cameraId}/mjpeg?v={version}`} />     // 原始画面
<img src={`/api/cameras/{cameraId}/model-mjpeg?task_mode={mode}`} />  // 标注画面
```

- 使用 `streamVersion` 参数强制浏览器在切换摄像头时重新加载（绕过缓存）
- 原始 MJPEG 无标注；模型 MJPEG 含检测框、ID、车牌、白名单状态、速度

### 3.2 ResourceMonitor（系统资源监控条）

底部固定展示 4 个仪表盘：
- CPU 利用率 + 物理/逻辑核心数
- 系统内存 (已用/总量 GB)
- GPU 利用率 + 显存 + 温度
- 推理耗时 (ms, 归一化到 [0,200] → [0%,100%])

### 3.3 功能中心（模态页）

- 侧边栏菜单 + 主内容区域
- 管理员可见 10 个页面，普通用户 5 个（历史/告警/账号/白名单/智能报告）
- 摄像头管理页根据 `CAMERA_TYPE_OPTIONS` 渲染新增表单
- 管理员“道路建模”入口根据 Vite `BASE_URL` 生成同源地址，支持根路径和子路径部署

### 3.4 验证码可靠加载

登录页把验证码重试逻辑从 JSX 提取到 `auth.js`。首次请求失败后依次等待 1 秒、2 秒和 4 秒，最多发起 4 次请求；任意一次成功即停止，全部失败时向登录页透传最后错误。加载期间显示明确状态，避免用户把网络延迟误认为验证码缺失。

### 3.5 语音控制

```javascript
// Web Speech API: SpeechRecognition
recognition = new webkitSpeechRecognition()
recognition.lang = "zh-CN"
recognition.continuous = false

// 指令解析:
"切换到道路异常"    → setTaskMode("road_anomaly")
"打开热力图"        → heatmapMode toggle
"打开历史记录"      → setFeatureView("history")
"当前车辆数"        → 语音朗读 analysis.traffic_stats.vehicle_count
```

## 4. 热力图模块 (`heatmap.js`)

### 4.1 模式解析

```javascript
function resolveHeatmapMode(camera) {
    const configured = camera?.heatmap_mode || "auto";
    if (configured !== "auto") return configured;
    // 手机/USB摄像头 → frame模式 (画面叠加)
    // 沙盘RTSP → road模式 (道路模型)
    return FRAME_CAMERA_TYPES.has(camera?.type) ? "frame" : "road";
}
```

三种模式：
- `off`: 不渲染热力图
- `overlay` (frame): 在原始画面上叠加归一化热点
- `schematic` (road): 在道路示意底图上绘制车道/路口拥堵色块

### 4.2 画面热点 (`frameHeatmapSpots`)

```javascript
// 从 AnalysisResult.detections 提取热点坐标
对每个检测框:
  centerX = ((x1+x2)/2 / source_width) * 100    // 归一化到 %
  centerY = ((y1+y2)/2 / source_height) * 100
  strength = confidence (0-1)
  size = 20 + confidence * 10
```

### 4.3 检测框变换 (`frameDetectionBoxes`)

将检测框坐标从像素空间映射到百分比坐标，供 Canvas 叠加渲染。

### 4.4 道路示意底图 (`schematicRoadBaseUrl`)

从 `/api/cameras/{id}/road-mask` 获取 `schematic_data_url` (PNG base64)，作为热力图 Canvas 的底图。

## 5. 前端状态管理

无 Redux/MobX 等第三方状态库，使用 React 原生 `useState` + `useEffect` 轮询：

```
token + user        ← POST /api/auth/login
cameras[]           ← GET /api/cameras
currentCameraId     ← 用户选择或默认 "live1"
taskMode            ← 用户切换 (traffic / road_anomaly)
analysis            ← 轮询 /api/analysis/latest
resources           ← 轮询 /api/system/resources
weather             ← 轮询 /api/weather/beijing
cameraStatus[]      ← 轮询 /api/cameras/status
heatmapData         ← 轮询 /api/road-model/heatmap
featureView         ← 用户点击功能中心菜单项
```

## 6. 数据流

```
后端                              前端
──                                ──
MJPEG stream ─────────────────→ <img src> 自动拉取
latest_result (内存) ←── 轮询 ── analysis state
system_monitor       ←── 轮询 ── resources state
heatmap_snapshot     ←── 轮询 ── heatmapData → Canvas渲染
weather              ←── 轮询 ── weather state
────────────────────────────────────────────
用户操作:
  切换摄像头 → POST /api/cameras/{id}/start
  切换模式   → 改变 taskMode → 改变 MJPEG URL
  处置告警   → PUT /api/incidents/{id}
  新增白名单 → POST /api/whitelist
```
