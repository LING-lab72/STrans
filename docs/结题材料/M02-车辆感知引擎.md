# M02：车辆感知引擎 — LocalModelService

## 1. 模块定位

`LocalModelService` 是 STrans 最核心的感知模块（`backend/app/services/local_model.py`，71 个符号，1473 行），负责从单帧图像中提取完整的结构化交通信息。它不仅运行目标检测，而是一套完整的"**检测→跟踪→过滤→OCR→决策→统计**"流水线。

## 2. 架构总览

```
LocalModelService
├── 状态内存 (跨帧持久)
│   ├── _models: dict[name → YOLO]         # 模型懒加载缓存
│   ├── _visual_track_memory                # 视觉轨迹: 框平滑+速度预测+元数据
│   ├── _track_plate_memory                 # 车牌轨迹: OCR文本→多帧稳定确认
│   ├── _track_motion_memory                # 运动状态: 世界坐标+速度样本
│   ├── _vehicle_gate_memory                # 门禁决策: 短时缓存防跳变
│   ├── _view_mode_memory                   # 视角: 近景/远景自动切换
│   └── _vehicle_count_history              # 车辆数: 5帧中值稳定
│
├── 模型
│   ├── YOLO11n (auto, 默认)
│   ├── YOLOv11s-VisDrone (visdrone, 远景)
│   ├── YOLOv8n (fallback, 保护模式)
│   └── HyperLPR3 (车牌OCR, 懒加载)
│
└── 核心入口
    ├── infer_jpeg(jpeg_bytes) → (AnalysisResult, annotated_jpeg?)
    └── infer_frame(np.ndarray) → (AnalysisResult, annotated_frame?)
```

## 3. 完整算法流水线 (`infer_frame`)

### 3.1 视角判断 (`_auto_small_target_mode`)

```
输入: camera_id, frame, model, frame_counter

1. 仅对 custom+ 宽高比≥1.5 的摄像头自动判断
2. 每 20 帧采样一次: YOLO(conf=0.15, imgsz=640) 探测
3. 有面积≥1.5%的大车? → near_votes++ → mode="near"
4. 无大车?          → far_votes++  → 连续2次→ mode="far"
5. 视角切换时 → _reset_camera_tracking_memory (清内存)
6. far模式: 裁剪画面中间区域 (22%~55%宽度), imgsz≥960
```

**设计意图**：远景画面中车辆极小（可能只有30×20像素），通过裁剪+提高分辨率让小目标在模型输入中相对变大。

### 3.2 检测与跟踪

```python
results = model.track(
    inference_frame,
    conf=min(conf, 0.08) if far else min(conf, 0.18),  # 保召回!
    imgsz=inference_size,
    tracker="bytetrack_sandtable.yaml",
    persist=True,     # 跨帧持久轨迹
    agnostic_nms=True,# 类别无关NMS: 防止同框多类
    device=device,
)
```

**关键决策——低阈值保召回**：
- 默认最小置信度：远景 0.08，普通 0.18（远低于常规 0.35-0.5）
- 召回优先，后续用道路 ROI、外观可信度、时序稳定等规则逐层排除误检

### 3.3 逐目标处理循环

对每个 YOLO 检测结果执行：

```
for box, class_id, conf, track_id in records:
    ├─ 1. 类别判断: is_vehicle / is_person
    ├─ 2. 道路ROI过滤: is_box_on_road(bbox)
    │     └─ 框底边82%处作为路面接触点 → pointPolygonTest
    │     └─ 底部被裁剪的框降级用中心点判断
    ├─ 3. 面积过滤: vehicle≥600px, person≥280px
    ├─ 4. ByteTrack ID 恢复: _recover_visual_track_id
    │     └─ 从近期(3/7/9帧)丢失的轨迹中按IoU匹配
    ├─ 5. 框平滑: _smooth_visual_box
    │     ├─ 微小位移(<3px): EMA 75/25 平滑
    │     └─ 较大位移: 速度预测 + 测量 72/28 融合 + 前推0.25帧
    ├─ 6. 外观可信度判断: _visual_vehicle_is_credible
    │     ├─ 大面积(≥3.5%): 对比度≥25 + 边缘密度≥0.022 → 可信
    │     ├─ 高置信(≥0.48): 对比度≥23 + 边缘密度≥0.032 → 可信
    │     ├─ 中等面积(≥1%): 边缘密度≥0.052 + 对比度≥24 + 熵≥3.2
    │     └─ 小面积: 边缘密度≥0.075 + 对比度≥23 + 熵≥3.25
    ├─ 7. 不可信 + 低置信(<0.22) → 跳过(省OCR)
    ├─ 8. OCR 预算控制(流模式):
    │     └─ plate_ocr_budget=2 → 每轮最多2次OCR, 顺序轮转
    ├─ 9. 轨迹车牌稳定: _stable_track_plate
    │     ├─ 多区域裁剪OCR投票 (3个band + 权重)
    │     ├─ 最近6帧观察, ≥2次相同文本 → 确认
    │     ├─ 首次识别置信度≥0.88 → 立即确认
    │     └─ 已确认车牌 → 75帧过期
    └─ 10. 白名单决策: _stable_decision
          ├─ 白名单车辆 → 缓存45帧
          └─ 非白名单 → 需同位置重复匹配才沿用旧决策
```

### 3.4 车辆外观可信度 (`vehicle_visual_features`)

```python
def vehicle_visual_features(crop):
    gray = cv2.cvtColor(crop, COLOR_BGR2GRAY)
    edges = cv2.Canny(gray, 55, 145)
    edge_density = countNonZero(edges) / edges.size
    contrast = np.std(gray)
    histogram → 32 bins → entropy
    return {edge_density, contrast, entropy}
```

**设计意图**：沙盘中的橡皮擦、瓶盖等非车辆物体可能被 YOLO 误识别为 car。真实模型车辆有车窗、货架、轮胎和车牌边缘，crop 内部具备更多局部结构（高边缘密度+高对比度+高熵），以此区分。

### 3.5 预测补充 (`_predicted_visual_tracks`)

漏检的目标做有限帧预测：

```
对每个不在 seen_track_ids 中的历史轨迹:
  ├─ 有车牌 → 最多预测 9 帧
  ├─ 可信车辆 → 最多预测 7 帧  (vehicle模式)
  │            → 最多预测 3 帧  (anomaly解释模式)
  └─ 普通目标 → 最多预测 3 帧
  confidence *= 0.91^missing_frames  (指数衰减)
  min_prediction_confidence ≥ 0.10/0.16
```

**设计意图**：短暂遮挡不应导致车辆"消失"，但也不无限预测（形成幽灵车）。

### 3.7 重复框抑制 (`suppress_duplicate_vehicle_detections`)

```
车辆框按优先级排序: track_id > plate > not_predicted > confidence > area
对每个候选:
  与已保留框比较:
    same_track 或 same_plate 或 likely_same_vehicle_box → 保留高优先级
likely_same_vehicle_box 条件(排队车不合并):
  IoU≥0.42 或 overlap_of_smaller≥0.64
  或 (水平重叠≥0.62 且 中心距≤min(h)*0.46 且 面积比≤3.2)
```

### 3.8 速度估计 (`_track_speed_cm_s`)

```
1. 框底边中心 → 透视变换 → 世界坐标 (厘米)
2. 标定矩阵: live3 四点标定
   image: (580,115) (1145,115) (1480,890) (245,890)
   world: (0,0) (40,0) (40,70) (0,70)
   道路区域 40cm×70cm
3. 同一轨迹相邻帧:
   distance_cm = sqrt((X2-X1)²+(Y2-Y1)²)
   raw_speed = distance_cm / elapsed
4. 死区过滤:
   像素位移 < max(4, hypot(w,h)*4%)
   世界位移 < 1.5cm
   → is_stationary → raw_speed = 0
5. 异常值 > 200cm/s → 拒绝
6. 最近5样本取中值
7. 连续2次静止 → 速度为0
```

## 6. 与异常模式的交互

`include_people=True` 时输出行人检测 → 供 RoadAnomalyService 作为合法道路使用者解释，避免把行人误判为异物。车辆监控模式不输出行人（`include_people=False`），保持统计纯净。
