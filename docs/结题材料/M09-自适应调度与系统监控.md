## 2. SystemMonitor

### 2.1 指标采集

```python
def snapshot(self, inference_ms=None):
    return {
        "timestamp": now_iso(),
        "cpu": {
            "usage_percent": psutil.cpu_percent(),     # 整体CPU利用率
            "physical_cores": cpu_count(logical=False),
            "logical_cores": cpu_count(logical=True),
        },
        "memory": {
            "usage_percent": virtual_memory().percent,  # 系统内存占比
            "used_gb": used / GIB,
            "total_gb": total / GIB,
        },
        "gpu": {
            "available": pynvml可用?,
            "name": GPU型号,
            "usage_percent": GPU利用率,
            "memory_usage_percent": 显存占比,
            "memory_used_gb", "memory_total_gb",
            "temperature_c": GPU温度,
        },
        "backend_process": {
            "memory_mb": 本进程内存(RSS),
        },
        "inference": {
            "latest_ms": 最新推理耗时,
        },
    }
```

### 2.2 降级设计

`_gpu_metrics()` 整个包裹在 `try/except` 中，GPU 不可用时返回 `{"available": False, ...}`——不影响服务运行。`__init__` 中调用 `psutil.cpu_percent(interval=None)` 做首次采样预热。

## 3. AdaptiveModelScheduler

### 3.1 六档策略

| 策略 | 触发条件 | 模型 | 尺寸 | 间隔 | 置信度 |
|---|---|---|---|---|---|
| **manual** | 调度关闭 | 原请求 | 原配置 | 原配置 | 原配置 |
| **quality** | 静态图 或 GPU余量充足 | 原请求 | ≥768–960 | ≥80–400ms | ≤0.28 |
| **anomaly** | 道路异常任务 | 原请求 | 768–960 | ≥180ms | ≤0.25 |
| **protect** | 显存≥88% / 内存≥92% / CPU≥92% | **fallback** | ≤512 | ≥350ms | 原配置 |
| **realtime** | GPU≥88% / 推理≥150ms | 原请求 | ≤640 | ≥180ms | 原配置 |
| **balanced** | 资源正常 (默认) | 原请求 | 原配置 | 原配置 | 原配置 |

### 3.2 决策优先级

```python
def choose(camera_id, task_mode, ...):
    1. 资源采样 (1.5s 缓存)
    2. if not enabled        → manual
    3. if is_static_image    → quality
    4. if road_anomaly       → anomaly
    5. if 显存≥88/内存≥92/CPU≥92 → protect
    6. if GPU≥88/推理≥150    → realtime
    7. if GPU余量充足         → quality
    8. else                  → balanced
```

### 3.3 滞回机制

```python
# 非protect策略切换: 8秒冷却
if previous and previous.profile != decision.profile
   and decision.profile != "protect"
   and now - last_changed < 8.0:
    return previous  # 拒绝切换
```

**设计意图**：
- 系统资源有瞬时波动（如短期 CPU 尖峰）
- 模型切换有开销（加载权重、预热）
- protect 不冷却——资源压力时立即降级

### 3.4 资源缓存

```python
if now - _resource_cache_at >= 1.5:  # 1.5秒刷新
    _resource_cache = system_monitor.snapshot(inference_ms)
```

资源采样调 `psutil` 本身有开销，1.5 秒缓存避免推理主循环每次调用都做系统调用。

## 6. 系统分析

### 6.1 调度策略的"安全网"角色

调度器不改算法结论的业务语义，只选择"用什么模型、多大分辨率、多快检测"。它保证了：
- 高负载时不崩溃（protect 降级）
- 静态图用高质量参数（quality 档）
- 异常任务保留小目标分辨率（anomaly 档）
- 低负载时自动恢复高质量（balanced→quality）

### 6.2 局限性

- 策略阈值（如 88% 显存、150ms 推理）基于经验，未做自动化校准
- 不感知多摄像头共享 GPU 的累积效应（只关心全局利用率）
- 模型切换（auto↔fallback）的加载延迟约 1-3 秒，期间推理中断
