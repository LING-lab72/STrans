# M03：道路异常识别 — RoadAnomalyService

## 1. 模块定位

`RoadAnomalyService` (`backend/app/services/road_anomaly.py`，37 符号，831 行) 负责在道路区域检测不应出现的物体。输出三类异常事件：

| 类型 | 触发条件 | 来源 |
|---|---|---|
| `road_obstacle` | 道路 ROI 内无法由合法使用者解释的局部变化 | 帧差 + 静态场景 |
| `road_pedestrian` | 道路区域内的行人 | YOLO person 检测 (由上层注入) |
| `road_damage` | 路面破损 (坑洞、裂缝) | 可选 YOLO 路损模型 |

## 2. 核心设计哲学：可解释的多线索融合

> "与其把所有异常都交给一个缺少沙盘标注数据的端到端模型，我们采用可解释的多线索融合——模型负责解释正常道路使用者，几何和时序规则负责发现无法解释的局部变化。"

```
每一层过滤 = 一种可解释的信号
├── 光流    → "相机在动吗？"
├── 变化比例 → "是不是光照突变？"
├── YOLO框  → "是正常车辆/行人吗？"
├── 近期记忆 → "是刚开走的车留下的空位吗？"
├── 颜色    → "颜色和路面差多少？"
├── 纹理    → "有足够的内部结构吗？"
├── 形状    → "是标线还是真实物体？"
└── 时序    → "连续出现多帧了吗？"
```

## 3. 架构总览

```
RoadAnomalyService
├── _states: dict[camera_id → CameraState]

│   ├── pending_obstacles         # 运动候选(待3帧确认)
│   ├── pending_static_obstacles  # 静态候选(待2帧确认)
│   ├── unstable_frames           # 不稳定帧计数
│   └── last_damage_at            # 路损模型冷却
│
├── _damage_models: dict[path → YOLO]
│   ├── RDD YOLO12s (优选)
│   └── Pothole YOLOv8 (备选)
│
└── 核心入口
    ├── analyze_jpeg(jpeg, base_result) → AnalysisResult

```

## 4. 主流程 (`analyze_frame`)

```
1. state = _states.setdefault(camera_id, CameraState())
2. roi_mask = _road_roi_mask(camera_id, shape)
3. gray = _preprocess(frame, roi_mask)
     CLAHE(2.0) → GaussianBlur(5×5) → bitwise_and(roi_mask)
4. anomaly_boxes = _frame_difference_candidates(...)    [运动分支]
5. static_boxes  = _static_scene_candidates(...)         [静态分支]
6. anomaly_boxes = _merge_candidates(anomaly, static)    [合并]
7. damage_boxes  = _road_damage_candidates(...)          [路损分支]
8. 构造 AnalysisResult (detections + events + raw)
9. _update_baseline(state, gray, no_anomaly)
     └─ 有异常: alpha=0.005, 无异常: alpha=0.03
```

## 5. 运动分支 (`_frame_difference_candidates`)

### 5.1 ROI 策略 (`_road_roi_mask`)

```
live3 / 宽≥高的custom → live3梯形ROI:
  poly = [(0.30,0.11), (0.60,0.11), (0.77,0.82), (0.13,0.82)]
其他 → 通用多边形:
  poly = [(0.18,0.08), (0.82,0.08), (0.96,0.96), (0.04,0.96)]
```

**专门扩展** (`_vivid_object_roi_mask`)：live3 远景方向保留更宽的 ROI 用于暖色小物体（如乒乓球），因为保守的交通 ROI 可能把停在消失点附近的小物体裁剪掉。

### 5.2 全局运动抑制

```python
# Lucas-Kanade 稀疏光流
points = goodFeaturesToTrack(previous, maxCorners=120, qualityLevel=0.015, minDistance=12)
next_points, status = calcOpticalFlowPyrLK(previous, current, points)
valid = status == 1 (至少12个有效点)
deltas = next_points - points
global_motion = median(norm(deltas))
# > 2.5px → 不稳定帧++
# ≥ 12帧 → 重建背景
```

### 5.3 背景差分 + 自适应阈值

```python
diff = absdiff(gray, baseline_gray)
diff = bitwise_and(diff, roi_mask)
threshold = max(28, mean(diff[roi]) + 2.2 * std(diff[roi]))
_, binary = threshold(diff, threshold, 255)
binary = MORPH_OPEN(5×5) → MORPH_CLOSE(5×5, iterations=2)
```

### 5.4 变化比例检测

```python
changed_ratio = countNonZero(binary) / roi_pixels
if changed_ratio > 0.085:
    # 可能是相机移动或光照突变 → 不输出异物
    unstable_frames++; 同上≥12帧→重建背景
```

### 5.5 轮廓过滤

```
面积范围: [max(350, w*h*0.00045), w*h*0.018]
宽/高: ≥10px
合法目标扣除: recent_legal_boxes(8帧) + 当前YOLO检测框
  扣除阈值: IoU>0.04 或 coverage>0.35
```

### 5.6 多帧确认

```python
# pending_obstacles 中待确认候选
# 匹配阈值: IoU ≥ 0.30
# hits ≥ 3 → 确认为 anomaly
# 2帧未匹配 → 清除
```

## 6. 静态图片分支 (`_static_scene_candidates`)

静态图片没有时间差，启用独立的三通道外观分析：

### 6.1 亮度通道（bright）

```python
threshold = max(158, percentile(roi, 82))  # 自适应亮阈值
binary: gray > threshold 且 在ROI内
→ MORPH_CLOSE(7×7) → MORPH_OPEN(3×3)
```

### 6.2 Lab 色度通道（chromatic）

```python
# 与路面主色的Lab色度距离
median_a, median_b = median(roi_lab[:,1]), median(roi_lab[:,2])
delta_a = a_channel - median_a
delta_b = b_channel - median_b
chroma_distance = sqrt(delta_a² + delta_b²)
threshold = max(15, percentile(chroma_distance[roi], 82))
→ 与路面颜色显著不同的物体
```

### 6.3 HSV 暖色通道（vivid）

```python
# 专为沙盘验收中常见的橙黄小物体(乒乓球等)
hue ∈ [4, 28], saturation ≥ 85, value ≥ 80
→ 使用更宽的 vivid_roi
→ MORPH_CLOSE(5×5) → MORPH_OPEN(3×3)
```

### 6.4 合法目标扣除

```
硬证据(hard_explained):
  YOLO检测 + 有车牌 → 从掩膜中直接擦除
软证据(soft_explained):
  YOLO检测 + 无车牌 + 外观满足可信条件(conf≥0.24, edge≥0.08, contrast≥30, entropy≥3.55)
  → 不擦除掩膜, 但在后续 IoU/coverage 判断中使用
车牌代理(plate_proxies):
  全帧OCR检测到的车牌 + 未归属到已有检测框
  → 从掩膜中擦除, 防止被误判为异物
```

### 6.5 外观过滤

每个候选区域通过多层过滤：

```
1. 面积范围: [w*h*0.0008, w*h*0.015] (vivid: 更宽的 [0.00025, 0.08])
2. 最小边长: vivid≥5% 画面, 其他≥2.5%
3. 宽高比: vivid [0.35,1.85], 其他 [0.30,1.35]
4. 道路距离: 中心点距ROI边界 ≥ w*4.5%(vivid: 1.2%)
5. 垂直位置: 中心y ∈ [16%, 84%]
6. 亮度: 非vivid mean<120 → 需色度填充率 ≥ 0.28
7. 轮廓填充率 ≥ 0.20
8. 对比度 ≥ 21 + 熵 ≥ 2.7
9. 低饱和度(<16) + 弱纹理 → 过滤
10. 白色占比: 饱和度<45 且 亮度>145 占比>0.32 + 低熵 → 车道线
11. 合法目标重叠: IoU>0.45(coverage>0.68)硬/ IoU>0.40(coverage>0.74)软
```

## 13. 系统分析

### 13.1 状态隔离

每个摄像头有独立的 `CameraState`，背景模型互不干扰。切换摄像头时通过 `reset(camera_id)` 清除旧状态。

### 13.2 稳定性保障

| 机制 | 作用 |
|---|---|
| 光流 | 相机移动 → 暂停输出 |
| 变化比例 | 光照突变 → 暂停输出 |
| 多帧确认 | 噪声/反光 → 需连续出现 |
| 合法目标解释 | YOLO检测 → 扣除 |
| 近期记忆 | 车辆离开空位 → 扣除 |
| 低权重背景更新 | 防止异常物体被吸收 |

### 13.3 性能考量

- 灰度处理 + 稀疏光流（120角点）→ 低 CPU 开销
- 破损模型 2.5s 冷却 → 避免每帧加载 YOLO
- 静态图分支仅在 `include_static_scene=True` 时运行
