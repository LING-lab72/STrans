# 答辩 PPT 内容提纲

建议控制在 14–16 页，主线是“问题—系统—算法—迭代—结果—价值”，不要逐页罗列所有接口。

## 第 1 页：封面

- 项目名称：基于多源视觉感知的智慧交通沙盘监测与分析系统
- 副标题：车辆感知、道路异常、空间映射与业务闭环
- 团队、指导教师、日期

## 第 2 页：背景与问题

- 多路视频来源异构；
- 沙盘车辆小、遮挡强、车牌尺寸小；
- 道路标线、灯光和阴影容易触发异常误报；
- 算法结果若没有历史、告警和证据，难以形成完整系统。

配图：沙盘全景 + 3 个问题标注。

## 第 3 页：目标与成果边界

- 核心目标：多源视频接入、车辆/车牌、道路异常、道路状态；
- 工程目标：可视化、权限、历史、处置、证据、调度；
- 明确当前成果是二维道路模型与视觉分析平台。

## 第 4 页：系统总体架构

使用 `component-architecture.svg`，展示视频源、CameraHub、算法、道路逻辑、SQLite、FastAPI、React 和道路建模工具。

一句话：系统形成“采集—识别—分析—处置—归档”闭环。

## 第 5 页：功能模块与用例

- 普通用户：监控、模式切换、历史、告警、报告；
- 管理员：摄像头、白名单、模型、调度、用户、审计；
- 配答辩版 UML 用例图 `use-case-overview.svg`；完整 `use-case.svg` 作为报告附图。

## 第 6 页：车辆感知主流程

展示车辆识别活动图 `vehicle-recognition-activity.svg`。

讲述顺序：远近视角 → YOLO/ByteTrack → 道路和外观过滤 → OCR → 白名单 → 速度/统计。

## 第 7 页：车辆与车牌关键优化

- 低阈值保召回、后置规则抑制误检；
- 视觉 ID 恢复、框平滑、短时预测；
- 车辆裁剪 OCR + 全帧 OCR；
- 车牌轨迹记忆与多帧稳定；
- 重叠车辆框和车牌反推框去重。

配图：同一张图的“原始检测—优化后—车牌/白名单”三联图。

## 第 8 页：道路异常算法

展示异常识别活动图 `road-anomaly-activity.svg`。

突出：道路 ROI、光流稳定性、背景差分、合法目标解释、颜色/纹理/形状和多帧确认。

## 第 9 页：道路映射与拥堵热力图

- 固定视角：单应矩阵 → 道路/路口 → 活跃轨迹热力；
- 移动视角：SegFormer 道路掩膜 → 画面热点裁剪；
- 数量 0.85 + 速度 0.15 的拥堵评分。

配图：原始画面、道路掩膜、道路示意热力图。

## 第 10 页：自适应模型调度

- 展示 `adaptive-scheduler-state.svg` 中的 quality / balanced / realtime / protect / anomaly 五档；
- 输入 CPU、内存、GPU、显存、推理耗时和任务类型；
- 强调缓存与 8 秒滞回。

## 第 11 页：软件工程与数据闭环

- 使用 `data-closed-loop-sequence.svg`，说明统一 `AnalysisResult`；
- SQLite 历史和告警；
- 告警处置状态；
- 原始帧/标注帧/JSON/SHA-256 证据包；
- RBAC、用户和审计。

## 第 12 页：算法迭代路线

使用 `algorithm-evolution-flow.svg` 横向时间轴：

> 视频闭环 → 核心感知 → 去重/OCR → 道路热力 → 道路语义约束 → 异常稳定性 → 自适应调度与证据链

每一阶段只写“问题—改进—效果”，提交号放在页脚作为证据。

## 第 13 页：真实识别结果

后续由 F 盘数据和 Playwright 截图补齐：

- 车辆/跟踪/速度/车牌/白名单例图；
- 道路异物、道路行人、路损例图；
- 道路掩膜与热力图；
- 每图标注数据名、模型、阈值、耗时和结果。

## 第 14 页：量化评测

- 检测 Precision / Recall / F1；
- 车牌检测率和整牌准确率；
- 异物事件级 Precision / Recall；
- FPS、P50/P95 延迟、CPU/GPU/显存；
- 至少给出一组错误案例及原因。

如果数据集没有真值，必须写明“演示样例统计”，不要伪装成严格精度评测。

## 第 15 页：创新点与项目价值

1. 面向沙盘的小目标和遮挡适配；
2. 车辆—轨迹—车牌—白名单联合识别；
3. 几何道路模型与视觉结果融合；
4. 可解释道路异常多线索融合；
5. 从算法展示到权限、证据和处置的完整闭环。

## 第 16 页：不足与展望

- 数据规模和真值标注仍需扩大；
- 非参考摄像头速度仍是估计；
- 道路异常依赖场景阈值，需进一步数据驱动；
- 后续可增加独立碰撞识别、信号灯识别和跨线流量统计；
- 完成更多摄像头标定与模型量化部署。

## 演示顺序建议

1. 登录并启动摄像头；
2. 展示车辆监控：框、ID、车牌、白名单、速度；
3. 展示道路/路口热力图；
4. 切换道路异常模式并展示异物；
5. 打开告警处置和证据包；
6. 展示历史、调度和智能报告。
