# M01：视频接入层 — VideoStreamService + CameraHub

## 1. 模块定位

视频接入层是 STrans 系统的数据入口，负责将异构视频源统一解码为 JPEG 帧供上层算法消费。由两个核心组件构成：

| 组件 | 文件 | 职责 |
|---|---|---|
| `VideoStreamService` | `backend/app/services/video_stream.py` | 单路流的解码、帧缓存、状态监控 |
| `CameraHub` | `backend/app/services/camera_hub.py` | 多路摄像头的配置管理、统一启停、MJPEG 分发 |

## 3. 算法 / 流程设计

### 3.1 VideoStreamService 读帧循环

```
start(source)
  │
  ├─ source 类型判断
  │   ├─ 纯数字 → cv2.VideoCapture(int)  (USB摄像头)
  │   ├─ rtsp:// → 设置 OPENCV_FFMPEG_CAPTURE_OPTIONS
  │   │            TCP传输 + nobuffer + low_delay
  │   ├─ http:// → cv2.VideoCapture(str)
  │   └─ 本地文件 → 解析为绝对路径, 判断后缀
  │       ├─ .jpg/.png/.bmp/.webp → is_static_image = True
  │       └─ .mp4/.avi 等          → cv2.VideoCapture + 循环
  │
  ├─ _read_loop(source, generation, stop_event)
  │   ├─ 静态图分支:
  │   │   cv2.imread → imencode JPEG → 写入 _latest_jpeg
  │   │   → 标记 is_static_image=True → 等待 stop_event
  │   │
  │   └─ 流分支:
  │       _open_capture → while True:
  │         ├─ cap.read() → imencode JPEG → 写入 _latest_jpeg
  │         ├─ 成功 → 更新 tracked FPS (1秒采样窗口)
  │         ├─ 失败 → consecutive_failures++
  │         │   ├─ < 5 → 保持旧帧, 等待 0.2s 重试
  │         │   └─ >= 5 → 跳出内层循环, 外层层重连
  │         └─ 本地文件: 控制帧间隔 (1/fps)
  │            实时流: 最快解码, 始终取最新帧
  │
  └─ 重连机制:
      指数退避? 固定 0.6s → 对 RTSP 短时断流反应快
```

### 3.2 关键设计决策

**a) 静态图特殊处理**：
- JPG/PNG 只有一帧有效数据。不做重连、不断流恢复，而是等待 stop 信号
- 标记 `is_static_image=True`，让上层（自适应调度、道路异常）切换处理策略
- 异常模式在静态图上启用 `_static_scene_candidates` 分支

**b) RTSP 低延迟参数**：
```python
os.environ["OPENCV_FFMPEG_CAPTURE_OPTIONS"] = (
    "rtsp_transport;tcp|fflags;nobuffer|flags;low_delay|max_delay;0"
)
```
- `rtsp_transport;tcp`：TCP 传输（可靠性优于 UDP），虽略增延迟但避免丢包花屏
- `fflags;nobuffer`：不预缓冲，立即输出帧
- `max_delay;0`：零额外延迟

**c) 地址脱敏**：
```python
def safe_source_label(source: str | None) -> str | None:
    if source is None or source.isdigit(): return source
    return "configured video source"  # 隐藏真实地址
```
日志、审计、状态 API 中的视频源地址全部替换为通用标签，防止 RTSP 凭据泄露。

**d) 代际隔离（generation token）**：
- 每次 `start()` 递增 `_generation`
- 读取线程每轮循环检查 `generation == self._generation`
- 旧代际线程即使未及时退出，其帧也不会写入 `_latest_jpeg`

### 3.3 CameraHub 多路管理

```
CameraHub.__init__
  ├─ _sources: 预置源(SANDTABLE_CAMERAS) + SQLite 自定义源
  ├─ _streams: 每个 source 对应一个 VideoStreamService
  └─ current_camera_id: 默认 "live1"

CameraHub 关键方法:
  ├─ start(camera_id): 非活跃流时才真正创建/启动
  │   └─ 检测 already running + same source → 直接返回 (幂等)
  ├─ stop(camera_id): _shutdown_locked → 释放 OpenCV capture
  ├─ latest_jpeg(camera_id): 返回该流的最新帧缓存
  ├─ mjpeg_frames(camera_id): Generator, MJPEG 边界帧
  └─ list_sources(): 内存字典 → List[CameraSource]
```

**预置源保护**：沙盘预置摄像头（type=sandtable）不可删除，只有自定义源可被管理 API 删除。

## 4. 数据模型

### CameraSource (`schemas/video.py`)
```python
class CameraSource(BaseModel):
    camera_id: str           # 唯一标识 (live1/live3/custom-xxx)
    name: str                # 显示名称
    type: CameraType         # custom | mobile | usb | sandtable
    stream_url: str          # RTSP/MJPEG/文件路径
    location: str            # 位置描述
    description: str | None  # 备注
    heatmap_mode: HeatmapMode # auto | off | overlay | schematic
    status: CameraStatus     # online | offline | connecting | error
    selected: bool           # 前端当前选中
```

### VideoSnapshot (`video_stream.py`)
```python
@dataclass
class VideoSnapshot:
    running: bool            # 是否在运行
    source: str | None       # 脱敏后的源地址
    connected: bool          # 是否成功连接
    frame_width: int | None
    frame_height: int | None
    frames_received: int     # 累计帧数
    fps: float               # 实际解码帧率
    is_static_image: bool    # 是否静态图片
    last_error: str | None   # 最近错误(已脱敏)
```

### 接口调用链

```
前端选择摄像头 → POST /api/cameras/{id}/start
  → CameraHub.start(id)
    → VideoStreamService.start(url)
      → 判断类型, _read_loop 后台线程启动
      → 解码, imencode JPEG, 写入 _latest_jpeg
  → 返回 VideoStatus

前端 <img src="/api/cameras/{id}/mjpeg">
  → CameraHub.mjpeg_frames(id)
    → VideoStreamService.mjpeg_frames()
      → 循环 yield: --frame + JPEG + \r\n
```

## 6. 系统分析

### 并发模型
- **CameraHub**：`threading.RLock` 保护 _sources 和 _streams 字典
- **VideoStreamService**：`threading.Lock` 保护帧缓存, `_lifecycle_lock` 保护启停状态机
- 每路流有独立后台线程运行 `_read_loop`，线程隔离

### 异常处理
```
帧读取失败(连续<5次) → 保持旧帧, 短等待重试
帧读取失败(连续≥5次) → 跳出内层, 外循环重连
RTSP断流 → 固定 0.6s 重连间隔
静态图源 → 不休眠等待新帧, 不重连
OpenCV异常 → 单独捕获, 标记离线, 重新进入重连
```

### 性能考量
- 帧缓存只有 1 帧（最新帧），无帧队列堆积
- 实时流以最快速度解码，确保推理拿到最新画面
- 本地文件按源帧率控制（`frame_interval`）
- MJPEG 端点间隔 `0.03s`（约 30 FPS 上限）
