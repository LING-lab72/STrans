# M10：算法客户端与辅助服务 — AlgorithmClient + WeatherService + sqlite_utils

## 1. AlgorithmClient

### 1.1 模块定位

`AlgorithmClient` (`backend/app/services/algorithm_client.py`，137 行) 是远程算法服务的抽象层，同时作为系统内部分析结果的总线。

### 1.2 双重角色

```
AlgorithmClient
├── 角色A: 结果总线 (始终活跃)
│   ├── latest_result: AnalysisResult   # 最新分析结果(内存缓存)
│   ├── events: list[TrafficEvent]      # 最近100条事件
│   ├── push_result(result)             # 本地模型结果推入
│   └── 事件去重: (type, camera_id, description) 三元组
│
└── 角色B: 远程服务客户端 (可选)
    ├── config: AlgorithmServiceConfig
    │   {enabled, base_url, health_path, infer_path, timeout}
    ├── health() → 遥测远程服务
    └── infer_frame(camera_id, jpeg) → 转发到远程推理
```

### 1.3 结果总线 (`push_result`)

```python
def push_result(self, result):
    normalized = result.model_copy(timestamp=...)
    self.latest_result = normalized

    # 事件去重: 前30条的三元组
    seen = {(type, camera_id, description) for event in events[:30]}
    new_events = [e for e in normalized.events if key(e) not in seen]

    self.events = new_events + self.events   # 新事件前置
    self.events = self.events[:100]          # 限制100条

    # 首次收到结果 → 状态变为 ready
    if self.status == "not_configured":
        self.status = "ready"
```

**设计意图**：
- `latest_result` 是前端轮询 `/api/analysis/latest` 的数据源——所有消费者共享一份缓存
- 事件去重防止同一障碍物在多帧中产生重复事件
- 100 条上限防止内存持续增长

## 3. sqlite_utils — 统一连接管理

### 3.1 模块定位

`sqlite_utils` (`backend/app/services/sqlite_utils.py`，27 行) 解决 BUG-001：Windows 上 SQLite 连接未关闭导致临时数据库无法删除。

### 3.2 实现

```python
class ClosingSQLiteConnection(sqlite3.Connection):
    """在提交/回滚后始终调用 close()"""
    def __exit__(self, exc_type, exc_value, traceback):
        try:
            return bool(super().__exit__(exc_type, exc_value, traceback))
        finally:
            self.close()  # ← 关键：无论提交还是回滚，都关闭

def open_sqlite(path) -> sqlite3.Connection:
    conn = sqlite3.connect(path, factory=ClosingSQLiteConnection)
    conn.row_factory = sqlite3.Row   # 统一返回字典行
    return conn
```

### 3.3 影响范围

`open_sqlite` 被以下 6 个服务统一使用：
- `CameraHub` (camera_sources)
- `AnalysisStore` (analysis_records, alert_incidents)
- `WhitelistStore` (vehicle_whitelist)
- `AuthStore` (system_users, system_sessions, audit_logs)
- `IntelligenceReportService` (intelligence_config, intelligence_reports)
