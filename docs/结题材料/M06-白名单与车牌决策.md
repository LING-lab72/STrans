# M06：白名单与车牌决策 — WhitelistStore + decide_plate

## 1. 模块定位

白名单模块 (`backend/app/services/whitelist.py`，27 符号，223 行) 负责车辆牌照的规范化、白名单持久化存储、模糊匹配查询和通行决策。通过 `decide_plate` 纯函数与前端、OCR 结果对接，输出 `allow/deny` 决策。

## 3. 车牌规范化 (`normalize_plate`)

```python
def normalize_plate(plate_no):
    plate = plate_no.strip().upper().replace(" ", "").replace("-", "")
    # 修复 HyperLPR3 的字符编码问题
    for broken, fixed in PLATE_PREFIX_FIXES.items():
        if plate.startswith(broken):
            plate = fixed + plate[len(broken):]
    # 车牌别名映射
    plate = PLATE_ALIASES.get(plate, plate)
    return plate
```

**修复映射**：
- `äº¬` → `京` (UTF-8 解码错误)
- `žŠ` → `京` (另一种编码错误)
- `京HF7912N` → `京H7912N` (OCR 多识别了一个字符)

## 4. 模糊匹配 (`match_plate`)

```python
def match_plate(self, plate_no):
    normalized = normalize_plate(plate_no)
    active = self.active_plate_nos()  # 启用状态的车牌列表

    # 1. 精确匹配 → 直接返回
    if normalized in active:
        return normalized

    # 2. 模糊匹配
    for candidate in active:
        score = SequenceMatcher(None, normalized, candidate).ratio()
        # 同省份(前2字符) + 同尾号(后4字符) → 提升置信度
        if same_region and same_tail:
            score = max(score, 0.92)
        if score > best_score:
            best = candidate

    # 3. 阈值 ≥ 0.78 返回最佳匹配
    return best if best_score >= 0.78 else None
```

**设计意图**：OCR 偶尔将 `0` 识别为 `O`、`8` 识别为 `B` 等。`SequenceMatcher` 的模糊匹配 + 同省份同尾号加分，容忍这类单字符错误。

## 5. 通行决策 (`decide_plate`)

```python
def decide_plate(plate_no, confidence=1.0):
    plate = normalize_plate(plate_no)
    matched = whitelist_store.match_plate(plate)

    if matched:
        return WhitelistDecision(plate, True, "allow", "白名单车辆，可通过")
    elif plate:
        return WhitelistDecision(None, False, "deny", "非白名单车辆，建议拦截")
    else:
        return WhitelistDecision(None, False, "deny", "未识别到车牌，需人工复核")
```

**关键设计**：
- `confidence` 参数保留但未在决策中使用——OCR 的置信度不稳定，已在 `_stable_track_plate` 层面处理
- `decide_plate` 是纯函数，无副作用，不访问数据库（通过全局 `whitelist_store` 间接调用）
- 输出是简单的三元判断：白名单/非白名单/未识别

## 6. 持久化设计

### 6.1 数据库表

```sql
CREATE TABLE vehicle_whitelist (
    plate_no TEXT PRIMARY KEY,   -- 规范化车牌
    owner TEXT NOT NULL,         -- 车主描述
    note TEXT NOT NULL,          -- 备注
    enabled INTEGER DEFAULT 1,   -- 启用状态
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);
```

### 6.2 初始化

系统启动时若白名单表为空，自动插入 5 辆默认白名单车辆：
```
京K9134J, 京E7654Z, 京H7912N, 京E4282Y, 京B6789T
```

### 6.3 缓存策略

```python
_active_cache: list[str]       # 启用状态的车牌列表
_active_cache_expires_at: float # 2秒过期
```

`active_plate_nos()` 方法缓存 2 秒，因为车牌匹配在每帧每个车辆上都会调用（高频操作），2 秒 TTL 在一致性和性能间取得平衡。

## 8. 与 LocalModelService 的集成

```
检测到车辆 + OCR成功
    ↓
_stable_track_plate() → 稳定车牌文本 (多帧确认)
    ↓
_stable_decision() → 查询 decide_plate()
    ↓
缓存到 _vehicle_gate_memory (白名单45帧, 非白名单同位置匹配)
    ↓
输出: DetectionBox.gate_action = "allow" / "deny"
       TrafficEvent: whitelist_pass / whitelist_block
```

**决策缓存**：
- 白名单车辆：缓存 45 帧（约 4-9 秒，取决于检测间隔）
- 非白名单车辆：仅当同位置重复匹配时才沿用旧决策
- 白名单表变更时：`clear_gate_memory(plate_no)` 清除相关缓存

## 9. 系统分析

### 9.1 安全考虑

- SQLite 白名单表受数据库文件权限保护
- 增删操作需 admin 权限，并有审计日志
- `decide_plate` 纯函数便于单独测试

### 9.2 扩展性

- `WhitelistDecision` 为 `@dataclass(frozen=True)`，不可变，线程安全
- `electronic_id` 字段预留电子标签识别（如 RFID）
- 决策函数独立于存储，未来可接入其他白名单源
